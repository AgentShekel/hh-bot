# hh-bot

Telegram-бот для автоматизации поиска и откликов на вакансии hh.ru с LLM-оценкой релевантности и автогенерацией сопроводительных писем.

## Что делает

- **Поиск** вакансий по настраиваемым фильтрам через браузерную автоматизацию (Playwright).
- **Pre-filter по title** (опциональный, по умолчанию пустой — добавляешь фрагменты названий ролей, которые off-target *именно для тебя*) — отсекает явные несовпадения ещё ДО вызова LLM, экономия токенов.
- **Оценка** релевантности каждой вакансии LLM-моделью — из коробки DeepSeek (основная) + Groq (fallback); обе работают по OpenAI-совместимому Chat Completions API, так что подключить другого провайдера — небольшая правка в `ai/llm_client.py`. Промпт анализатора generic — оценивает соответствие профилю кандидата из `prompts/analyzer_summary.txt` и содержит встроенную инструкцию, как ужесточить скоринг (role-type правила, анти-домены, базовые баллы) под свой профиль.
- **Маршрутизация** по баллу (пороги настраиваемые):
  - **≥ `SCORE_AUTO_APPLY`** (по умолч. 90) → автоотклик с автогенерируемым письмом — *только* при `AUTO_APPLY_ENABLED=true` (по умолчанию выключено).
  - **средний диапазон** → присылается в Telegram на ручное решение (кнопки «Откликнуться / Пропустить / Открыть на hh.ru»).
  - **< `SCORE_AUTO_SKIP`** (по умолч. 10) → автопропуск.
- **Cover letter** генерируется по твоему профилю в `prompts/candidate.txt`, проходит adversarial self-critique (regenerate при выдуманных числах, HR-клише или off-tone формулировках).
- **Отклик по ссылке**: кидаешь ссылку на вакансию hh.ru прямо в чат — бот открывает её, оценивает, генерирует сопроводительное и показывает с кнопками «Отправить / Переписать / Отмена», не дожидаясь пока автопилот её найдёт. Понимает региональные поддомены и tracking-параметры; не отправляет повторный отклик.
- **Автопилот**: фоновый цикл поиска (темп подобран так, чтобы не словить анти-бот hh.ru) + периодическая сводка в Telegram.
- **Inbox-монитор** (`messages_loop`): раз в 5 минут проверяет негоциации hh.ru, классифицирует входящие как `TEST_TASK` / `QUESTION` / `MESSAGE`, пингует в Telegram.
- **Чёрный список компаний** (whole-word match), порог рейтинга работодателя, фильтр «только удалёнка», кросс-городовая дедупликация.
- **Telegram channel monitor** (опционально): читает публичные каналы с вакансиями через веб-зеркало (`t.me/s/<канал>`, без API-ключей), извлекает и оценивает посты, шлёт релевантные в Telegram — параллельно с hh.ru.

## Дисклеймер

> Использование автоматизации на hh.ru может нарушать пользовательское соглашение и приводить к блокировке аккаунта. Используйте на свой страх и риск. Бот написан как pet-проект для одного пользователя — не предполагается мульти-юзер деплой и не проходил юридической экспертизы.

## Стек

- **Python 3.10+** (нужен `Browser | None` синтаксис)
- `aiogram 3.x` — Telegram-бот
- `playwright` — автоматизация браузера hh.ru
- `httpx[socks]` — HTTP-клиент с поддержкой SOCKS5-прокси
- `python-dotenv` — конфиг
- `sqlite3` (встроен в Python) — хранилище

## Установка

```bash
git clone https://github.com/<YOUR_USERNAME>/hh-bot.git
cd hh-bot

python -m venv .venv
# Windows:
.venv\Scripts\activate
# macOS/Linux:
source .venv/bin/activate

pip install -r requirements.txt
playwright install chromium
playwright install msedge   # для интерактивного логина — открывается видимое окно Edge
```

## Настройка

Скопируй `.env.example` в `.env` и заполни значения:

```bash
cp .env.example .env
```

### Минимальный `.env`:

```
TELEGRAM_BOT_TOKEN=...           # получи у @BotFather
TELEGRAM_ADMIN_ID=...            # твой Telegram user_id (узнай у @userinfobot)
HH_LOGIN=+79001234567            # номер телефона для hh.ru

DEEPSEEK_API_KEY=...             # основная LLM (platform.deepseek.com)
GROQ_API_KEY=...                 # бесплатный fallback (console.groq.com)

RESUME_FILE=resume.txt
```

### Резюме

Положи `resume.txt` в корень проекта — обычный текстовый файл с твоим резюме. Бот будет использовать его как контекст кандидата при оценке вакансий и генерации сопроводительных.

### LLM-провайдер

Из коробки бот использует **DeepSeek** как основную модель и **Groq** (бесплатный тариф) как fallback, если DeepSeek недоступен или ключ не задан. Обе — по OpenAI-совместимому Chat Completions API.

1. DeepSeek: зарегистрируйся на https://platform.deepseek.com, создай ключ, задай `DEEPSEEK_API_KEY`.
2. Groq (опциональный fallback): https://console.groq.com, создай ключ, задай `GROQ_API_KEY`.

Чтобы подключить другого провайдера — поправь клиент в `ai/llm_client.py`.

### Прокси (опционально)

`HH_PROXY` — прокси для браузера hh.ru (нужен при запуске вне РФ — hh.ru блокирует иностранные IP). `TG_HTTP_PROXY` — прокси для веб-зеркала Telegram, если `t.me` заблокирован в твоей сети.

```
HH_PROXY=http://user:pass@host:port
```

## Запуск

```bash
python main.py
```

При первом запуске бот скажет, что нужна авторизация. В Telegram отправь боту:

```
/login
```

Откроется видимое окно браузера Edge — залогинься на hh.ru вручную (капча, СМС, что угодно). Бот автоматически подхватит сессию и переключится в headless-режим.

## Команды бота

| Команда | Что делает |
|---|---|
| `/start` | Статус, список команд |
| `/login` | Открыть браузер для интерактивной авторизации на hh.ru |
| `/addfilter` | Добавить фильтр поиска (название, ключевые слова, город, ЗП от, опыт) |
| `/filters` | Список активных фильтров |
| `/search` | Запустить ручной поиск по всем фильтрам |
| `/autopilot` | Запустить один цикл автопилота вручную |
| `/recheck` | Переоценить вакансии без оценки |
| `/stats` | Статистика откликов |

## Безопасность

- Доступ ко всем командам ограничен по `TELEGRAM_ADMIN_ID` — бот написан под одного пользователя.
- Cookies hh.ru сохраняются в `hh_cookies.json` (в `.gitignore`).
- Описания вакансий изолируются маркерами в LLM-промте — частичная защита от prompt injection через текст вакансии.
- Все секреты — через `.env`, ничего не хардкожено в исходниках.

## Telegram channel monitor (опционально)

Бот может собирать вакансии из публичных Telegram-каналов
(`@datasciencejobs`, `@workinai`, `@prog_jobs` и т.п.) параллельно с
hh.ru. Каналы читаются через **публичное веб-зеркало**
(`https://t.me/s/<канал>`) — без API-ключей, без my.telegram.org, без
логина и session-файлов.

Настройка:

1. Скопируй `data/tg_channels.example.yaml` в `data/tg_channels.yaml`,
   укажи нужные каналы и подкрути `role_filter` под свой профиль.
2. (Опц.) Прогон теста: `python -m parser.tg_client test` — покажет
   последние посты каждого канала, ничего не запуская.
3. Запусти бота — монитор стартует автоматически и раз в `TG_PULL_INTERVAL`
   (по умолч. 30 мин) шлёт релевантные посты в Telegram с кнопками
   «Открыть в Telegram / Черновик письма / Пропустить».

Монитор только уведомляет (в Telegram отклик — это личное сообщение
рекрутёру), поэтому работает и в `MANUAL_MODE`. Если `t.me` заблокирован
на твоей сети — задай `TG_HTTP_PROXY` в `.env`.

## Известные ограничения

- **Селекторы hh.ru захардкожены** — при редизайне UI может потребоваться обновление `parser/hh_client.py`.
- **Сетка городов** в `_resolve_area` — только 15 крупнейших. Остальные ищутся без фильтра по региону (с предупреждением в логе).
- **БД синхронная** — `sqlite3` блокирует event-loop на больших объёмах. Для одного пользователя норм.
- **Нет тестов** на бизнес-логику.
- **TG-монитор** только уведомляет — автопилот не подаёт отклики на TG-вакансии (в Telegram отклик это личное сообщение рекрутёру); они собираются для ручного просмотра и черновика письма.
